Angular Components

A Controller...
  • is used to set up the initial state of the $scope and add behavior to it.
  • shouldn't try to do too much. It should only apply business logic for a single view.
  • manages $scope properties and makes them available to a HTML template where the controller is registered using the ng-controller directive. Where it is registered, Angular will instantiate a new Controller object.
A Directive...
  • contains specific behavior to manipulate a custom DOM element.
  • is used to separate DOM manipulation from the views and assign reusable functionality to elements.
  • is a standalone reusable component that can be safe and easy to add any number of times throughout the application. It is added to the HTML as an element or attribute.
  • reduces redundant code by isolating data for specific instances. If it has a scope, it is not meant to be shared. Otherwise it inherits the scope of its parent. It can have associated HTML templates where it replaces it's own specific instances information, reducing redundancy in the HTML.
A Service...
  • contains reusable business logic independent of views.
  • is used when you need to share data across different components.
  • is a singleton, it is created once and injected as a dependency when needed.
  • ensures you are dealing with the correct data by managing its access and modification throughout the application.
